All About Github

  • Q. What is a version control system?
  • A. Version control systems are tools that help teams manage changes to code over time.
    • Q. What is a Git?
    • A. Git is a tool made to manage source code.
      • Q. What is Github?
      • A. Github is an online tool made to manage source code over several computers.
        • Q. What is the difference between Git and Github?
        • A. Git is a version control system that helps organize source code, while Github takes those groups of codes, also called repositories, and makes them puplic on the internet.
          • Q. Who started Github and how was it started?
          • A. Github was started by Tom Preston-Werner, Chris Wanstrath, P.J. Hyett, and Thomas Dohmke. It was started as a fun project between the four that snowballed into what it is now.
            • Q. What company owns it now?
            • A. Microsoft Corp now owns Github.
              • Q. How much does a Github acct cost?
              • A. Github offers one free repository, however there is a premium account for $49 per month.
                • Q. What is the Octocat?
                • A. Octocat is Github's mascot, it has the body of an ctopus with a cat head on top.

                  Git/Github terms:

                  • Repository- a central storage area for files and resources such as images.
                  • Commit- the core building block units of a coding project.
                  • Fork- copy of existing repositories where the new owner disconects the codebase from previous committers.
                  • Push- used to locally upload repository content to a separate repository.
                  • Pull requests- notifies others about changes you have made to a branch in a repository.
                  • Workflows- a recommendation on how to use Git/GitHub to accomplish work in a consistent timeframe.
                  • Issues- items you can create in a repository to plan out and track work thats being completed.
                  • Raw button- opens the file in raw form, making any html format dissapears.
                  • Blame button- examines the authors of each line and edit made by committers. Holds authors of each piece of code accountable.
