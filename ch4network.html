Chapter 4 Networking & The Internet

Network Classifications

  • LAN- Local Area Network. Usually a collection of computers in a single building or complex.
  • MAN- Metropolitan Area Network. Computers connected in an intermediate are, such as a local community.
  • WAN- Wide Area Network. Links machines over a greater distance. Could be across town or across the world.

Network Models

  • Bus Network- Method of computer communication where all machines are connected to a central communication line.
  • Star Network- Communication method where all machines are connected to a central computer or server.

Protocols

  • CSMA/CD Ethernet- Carrier Sense, Multiple Access with Collision Detection. Protocol that dictates that each message be broadcast to all the machines on the bus. Each machine only keeps what data is addressed to itself.
  • CSMA/CD Wifi- Carrier Sense, Multiple Access with Collision Avoidance. Focuses on avoiding two machines transmitting at one time (collisions).

Combining Networks:

  • Repeater- Passes signals back and forth between buses. Does not consider the meaning of the signals.
  • Bridge- A more complex repeater. Connects two buses, but takes the destination into consideration and only sends a message across the connection only when that message is destined for a computer on the other side. More efficient than a repeater.
  • Switch- Essentially a bridge with multiple connections, allowing it to connect several buses rather than just two. Still considers the destination of the signal in order to minimize traffic across spokes.
  • Router- Provides links between networks while allowing each network to maintain its unique internal characteristics.
  • Gateway- The point at which one network is linked to an internet. This is because it serves as a passageway between the network and the outside world.

Client/Server Model

  • P2P: Peer to Peer- Involves processes that provide service to and receive service from each other.
  • Distributed System- Many modern software systems, such as global information retrieval systems, company-wide accounting and inventory systems, computer games, and even the software that controls a network's infrastructure itself are designed as distributed systems, meaning that they consist of software units that execute as processes on different computers.
  • Cloud Computing- Cloud computing, whereby huge pools of shared computers on the network can be allocated for use by clients as needed.

Internet Adressing

  • Teir 1- ISPs that consist of very high-speed, high-capacity, international WANs.
  • Teir 2- ISPs that tend to be more regional in scope and less potent in their capabilities.
  • Teir 3- Access or tier-3 ISP. An access ISP is essentially an independent Internet, sometimes called an intranet, operated by a single authority that is in the business of supplying Internet access to individual homes and businesses.

IP Adress

  • ICANN
  • DNS- Domain Name System, translates domain names into IP addresses so browsers can load internet resources.
  • TLD: Top-Level Domain- As an example, the domain name of Marquette University is . Note the suffix following the period. It is used to reflect the domain's classification, which in this case is “educational,” as indicated by the suffix. These suffixes are called top-level domains (TLDs).
  • Country-Code TLDs

Internet Protocols

  • FTP- An application for accessing another computer from a great distance used the Telnet protocol, or later, the Secure Shell (SSH) protocol.
  • HTTP- As webservers and browsers have become more sophisticated, more and more of these traditional network applications have come to be handled by webpages via the powerful Hyper Text Transfer Protocol.
  • SMTP- Simple Mail Transfer Protocol defines a way that two computers on the network may interact when transmitting an email message from one host to the other.

World Wide Web

  • URL- In order to locate and retrieve documents on the Web, each document is given a unique address called a Uniform Resource Locator (URL).
  • Client-Side Activities- Activities if they are performed by a client (such as a browser).
  • Server-Side Activities- Activities if they are performed by a server (such as a webserver)

Internet Four-Layer Software

  • Application Layer- A message typically originates in the application layer.
  • Transport Layer- Sequence numbers to the small segments it produces so that the segments can be reassembled at the message's destination.
  • Network Layer- Decides in which direction a packet should be sent at each step along the packet's path through the Internet.
  • Link Layer- Has the responsibility of transferring the packet.

Internet Is:

  • Packet Switching Network- Transfers small pieces of data across various networks.
  • Fault Tolerant- Enables the operationg system to respond to a failure in hardware or software.
  • Rdedundant- Keeping data in multiple places within a database or storage system.

Cybersecurity

  • Malware- Software meant to harm the computer.
  • Virus- Inserts itself into programs already existing in the machine.
  • Worm- Resides in computers and, through a network, forwards copies of itself to other computers.
  • Trojan Horse- Disguises itself as a desireable program, only to perform other activies not intended by the user.
  • Spyware- Collects information about activities on the computer and reports that information back to the instigator of the attack.
  • Phishing- Obtaining information by asking for it, typically for a false purpose or promise.
  • DDos Attack- A denial of service attack spread over many hosts.

Protection

  • Firewall-Filters traffic passing through a point on the network.
  • Spam Filter- Firewalls designed to block unwanted email.
  • Proxy Server- A proxy server is a software unit that acts as an intermediary between a client and a server with the goal of shielding the client from adverse actions of the server.
  • Antivirus Software- Used to detect and remove the presence of known viruses and other infections.
  • Cryptography- The study of sending and receiving messages securely in the presence of adversaries.
  • Public Key Encryption- Involves techniques by which the encryption and decryption algorithms are known, and one of the keys is known, but the encoded message remains secure.
